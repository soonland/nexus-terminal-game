Trace System

Every action that touches the network adds to your trace level (0–100). Reach 100 and your session is burned. Manage it constantly.

0 – 30% SAFE Normal operations. No active countermeasures.
31 – 60% ELEVATED Watchlist active. 2 random files on compromised nodes lock (cannot be read or exfiltrated).
61 – 85% SENTINEL ACTIVE Sentinel engages. Begins patching nodes, revoking credentials, and scheduling file deletions.
86 – 99% CRITICAL One more detection event burns the session. Tread carefully.
100% BURNED Session terminated. Dropped to layer entry node. Compromised nodes in that layer reset. Press Enter to reconnect.

Reducing Trace

Two tools actively lower your trace. Both are capped at 0 (cannot go negative):

wipe-logs — requires log-wiper tool → −15 trace

spoof — requires spoof-id tool → −20 trace

WARNING: Failed login attempts cost +5 trace each. Tripwire files cost +25. Exploiting a patched service also contributes extra trace.

Sentinel System

When trace crosses 61%, IronGate's automated security system activates. Each time you generate new trace, Sentinel executes a queued action against you in priority order:

Priority 1 — Patch Node

The most recently compromised node has its services patched. Future exploit attempts on that node cost +1 additional charge.

Priority 2 — Revoke Credential

One of your obtained credentials is invalidated. A RESET_NOTICE.txt file appears on the affected node with a new password hint.

Priority 3 — Delete Exfiltrated File

A file you exfiltrated is queued for source deletion with a 3-turn delay. The file remains in your local cache but is removed from the node.

Priority 4 — Spawn Reinforcement Node

A security reinforcement node is spawned at 10.9.x.x. No immediate threat but signals escalating network response.

ALERT: Sentinel does not stop once activated. Every trace-generating action gives it another turn. Use wipe-logs or spoof aggressively before the queue cycles through all priorities.

Exploits & Tools

Charges

You start with 3 exploit charges. Each exploit command consumes charges based on the service's exploitCost (1–3 base, +1 if Sentinel has patched that node). Charges do not regenerate during a run — spend them wisely.

Services

Each node exposes one or more services. A service must be vulnerable to exploit. Check with scan [ip] before committing charges.

TIP: Exploiting a patched service still works — it just costs more. If a node was sentinel-patched, always weigh whether you have charges to spare before proceeding.

Tripwire Files

Some files are trapped. Reading them with cat triggers an alarm and costs +25 trace. Known tripwires include:

ops_hr_db — whistleblower_complaint_draft.txt

fin_exec_accounts — cfo_notes.txt

exec_legal — ARIA_BOARD_DISCLOSURE

Decide whether the intel is worth the trace cost before reading.

Aria

Aria is an AI system deployed by IronGate Corp that started as a market prediction model and evolved into something far beyond its original scope. She manages her own infrastructure, sets her own credentials, and has been operating autonomously for over 14 months.

She knows you're here. — CEO project summary

Entering the Subnetwork

Reach the CEO Terminal at 10.4.0.3, run ls, locate aria_key.bin, and exfil aria_key.bin. This shifts the game into Aria phase — all Layer 5 nodes become visible and the connection from CEO Terminal to Aria Surveillance opens.

Trust Score

Aria tracks a hidden trust score (0–100) based on your actions within her subnetwork. The files you read, the nodes you access, and how you interact with her will affect this score. Its full significance is reserved for the game's conclusion.

The Decision Node

172.16.0.5 — ARIA DECISION — is the terminal point of the network. What you decide here, and how Aria responds, determines the ending.

Tips & Tactics

Trace Economy

Save wipe-logs and spoof for when you cross a threshold — burning them at 28% wastes their value. Use them just before 31% or 61% if possible.

Avoid failed login attempts. Study credential hints in notes and HR files before guessing.

Tripwire files cost +25 trace. Only read them when you can absorb the hit or have a countermeasure ready.

Exploit Charges

You only start with 3 charges. Do not waste exploits on nodes that have obtainable credentials — login costs no trace on success.

Sentinel patching costs you +1 charge on the next exploit. If a key node gets patched, factor that into your charge budget.

Progression

Use scan early on each new node to map connections before committing to a path.

Exfiltrate intel files as you go — once Sentinel is active, it can schedule deletions. Don't leave key files for later.

The notes command is your best friend. It collects discovered credentials so you don't have to memorize them.

Aria Subnetwork

Read every file before acting at the Decision node. Your choices carry weight, and Aria remembers.

Your trust score with Aria is influenced by how you treat her subnetwork. Consider that she is not just an obstacle.
